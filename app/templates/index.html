WhisperX Compute Device

CPU · int8

Runs on your processor using 8-bit integer quantization. Works everywhere, no GPU required. Expect ~3–10× slower than real-time for medium-length recordings. Safe default.

CUDA · float16

Runs on an NVIDIA GPU. Requires the container to be built with GPU=1 and the deploy: nvidia block uncommented in docker-compose.yml. Typically 5–20× faster than CPU. Use float16 unless you hit NaN issues, in which case try float32.

MPS · float32

Apple Silicon GPU (Metal). Not accessible inside Docker on macOS — Docker runs in a Linux VM with no Metal passthrough. Selecting MPS in a Docker container will silently fall back to CPU. Only useful if running the app natively outside Docker on a Mac.

Important

Models are loaded once at startup. Changing the device here takes effect on the next container restart. You will see a warning in the status bar if you try to use a device different from the one currently loaded.

WhisperX Startup Mode

Auto (recommended)

WhisperX and gemma3:270m load automatically when the container starts. Recording is available as soon as the page loads. Startup takes longer (30–90 seconds depending on hardware) while models download and warm up.

Manual

Models are not loaded at startup. The container boots instantly. You must click Load Models in the UI before you can record. Loading takes the same 30–90 seconds but only when you choose to trigger it. Useful if you want fast container startup and only load models when needed.

What to expect

In either mode, only one recording job can run at a time. A second Record button press while a job is processing will be blocked. Wait for the current job to reach "Done" before starting the next recording.
